# 西经东译播客自动生成器 - 使用指南

## 快速开始

### 1. 准备工作

确保安装了必需工具：
```bash
# macOS
brew install ffmpeg yt-dlp

# 其他系统请访问官网下载
# ffmpeg: https://ffmpeg.org/
# yt-dlp: https://github.com/yt-dlp/yt-dlp
```

### 2. 配置播客信息

编辑 `start_prompt.md` 文件：

```markdown
播客名字是{The new AI growth playbook for 2026 | How Lovable hit $200M ARR in one year}，嘉宾是{Elena Verna}。

YouTube URL: https://www.youtube.com/watch?v=xxxxx

当前要转译的cookie文件目录地址：{vedio_cookies/xxx.txt}

音色配置: speaker_0:{Deep_Voice_Man}, speaker_1:{Chinese (Mandarin)_Crisp_Girl}
```

### 3. 生成播客

```bash
# 完整流程（下载→ASR→翻译→TTS→合并）
python generate.py

# 或者只生成播客（如果已有ASR文件）
python auto_generate_podcast.py
```

## 重要安全机制

### ✅ ASR Hash验证
- 每个翻译文件都记录了源ASR的hash
- 加载翻译时自动验证hash是否匹配
- **如果不匹配，自动删除旧翻译并重新生成**

### ✅ ASR文件选择
- 优先使用今天生成的ASR文件
- 如果使用非今天的文件，会显示警告：
  ```
  ⚠️  未找到今天的ASR文件，选择最新的: xxx.json
    修改时间: 2026-01-01 15:30
    ⚠️  请确认这是正确的播客！
  ```

### ✅ 音频段隔离
- 不同播客的音频段保存在独立目录（按ASR hash）
- 避免断点续传时混用不同播客的音频

### ✅ SSL重试机制
- TTS生成遇到网络错误会自动重试3次
- 重试间隔：4秒、8秒、16秒（指数退避）

## 文件结构

```
output/
├── asr_results/               # ASR转录结果
│   └── qwen_asr_20260102_153544.json
├── translations/               # 翻译文件（含hash验证）
│   └── qwen_asr_20260102_153544_translations_enhanced.json
├── audio_segments/            # 音频段（按ASR hash隔离）
│   ├── final_abc123/          # 播客A的音频段
│   └── final_def456/          # 播客B的音频段
├── podcast_final/             # 最终播客
│   └── {播客名}.mp3
└── description/               # 播客描述文案
    └── {播客名}.txt
```

## 常见问题

### Q: 如何确保不会用错ASR或翻译文件？

**A: 系统现在有三层保护：**
1. **ASR选择**：优先今天的文件，非今天会警告
2. **翻译验证**：加载翻译时验证ASR hash，不匹配自动重新生成
3. **音频隔离**：不同播客的音频段完全分开存储

### Q: 如果翻译失败了怎么办？

**A: 删除翻译文件，系统会自动重新生成：**
```bash
rm output/translations/qwen_asr_*_translations_enhanced.json
python auto_generate_podcast.py
```

### Q: 如何清理旧播客的文件？

**A: 可以安全删除以下目录：**
```bash
# 清理所有中间文件（保留ASR）
rm -rf output/translations/*
rm -rf output/audio_segments/final_*
rm -rf output/temp/*
```
